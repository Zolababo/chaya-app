# 영업용 UI 패스 (한국 시장 기준)

점주 영업·매장 파일럿 전 **손님 `/t` · 점주 `/m`** 화면을 “한국에서 익숙한 QR 메뉴판” 수준으로 맞추기 위한 체크리스트입니다.  
기능·다국어·주문 파이프라인은 `docs/MVP_SALES_READY_202606.md` 를 전제로,**겉모습·동선·신뢰감**만 다룹니다.

---

## 1. 한국에 “표준 포맷”이 있나?

**하나의 공식 표준이 있는 것은 아닙니다.** 다만 아래는 사실상 **관습 표준(de facto)** 으로 많이 씁니다.

| 구분 | 내용 | CHAYA에 적용 |
|------|------|----------------|
| **법·접근성** | 공공·대형 서비스는 [KWCAG](https://www.wa.or.kr/)·모바일 터치 **최소 44×44px** 를 많이 참고 | 이미 `min-h-[44px]`·skip link 등 반영 중 (`a11y:baseline`) |
| **가격 표기** | **원(₩)**, 천 단위 콤마 `12,000원` 또는 `₩12,000` | `formatConsumerMoney` — 로케일별 통일만 유지 |
| **주문 결제** | 테이블 QR·무인 주문은 **“주문만 앱, 결제는 카운터”** 문구가 흔함 | `ConsumerOfflinePaymentCallout` — 영업 시 반드시 보여줄 것 |
| **레이아웃** | **가로 카테고리 칩** + **메뉴 리스트(썸네일·이름·가격·담기)** 또는 **2열 카드 그리드** | 현재: **리스트형** (`MenuBoard`) — 한국 앱들도 리스트 많음(배민·키오스크 2차 화면) |
| **하단 네비** | **메뉴 / 장바구니(뱃지) / 주문** 3탭 고정 | `BottomNav` — 업계 관습과 일치 |
| **품절** | 회색·딤·「품절」뱃지, 담기 버튼 비활성 | 구현됨 — **대비·크기**만 영업용으로 강화 |
| **외국인** | 상단 **언어 칩**(한·영·日·中) | `LocaleSwitcher` 8종 — **눈에 띄게**만 다듬으면 됨 |
| **점주** | 손님 화면과 **같은 카드**로 미리보기(WYSIWYG) | `MerchantMenuCatalog` — **간격·타이포** 맞추기 |

**레퍼런스(느낌만, 복제 목표 아님)**  
국내 테이블 QR·주문: 캐치테이블 매장 주문, 요기요 QR, 각종 POS 연동 QR 메뉴, 키오스크 2번째 화면(리스트+이미지).  
해외 Stitch 벤토 그리드는 **선택** — `docs/STITCH_TO_APP_MAP.md` §2.1.

**CHAYA 포지션:** “배달앱 clone”이 아니라 **「QR → 메뉴 → 담기 → 주방 전달 → 카운터 결제」** 가 10초 안에 이해되면 영업 통과.

## 2. 영업용 UI 패스 — 우선순위

완료 시 `[x]` 로 표시. **P0 = 영업 전 필수**, P1 = 파일럿 1곳 전, P2 = 여유.

### P0 — 손님 `/t/{tenant}` (QR 첫인상)

| # | 항목 | 한국 관습 | CHAYA 현재·목표 |
|---|------|-----------|----------------|
| P0-1 | **첫 화면 3초** | 매장명·테이블(있으면)·언어·메뉴가 바로 보임 | 헤더·FLOW 3단계 배너 정리, 언어 칩 시각적 강조 |
| P0-2 | **카테고리 칩** | 가로 스크롤, 선택=채움색 | 있음 — **패딩·스크롤 힌트**(그라데이션 페이드) |
| P0-3 | **메뉴 행** | 썸네일 56~72px, 이름 1줄, 가격 굵게, **담기 44px+** | 리스트 있음 — **담기를 primary(주황)**, 품절은 회색만 |
| P0-4 | **가격** | `12,000원` 정렬(tabular), 할인 시 취소선(선택) | `formatKrw` / locale — **자릿수 정렬** |
| P0-5 | **품절** | 담기 대신 「품절」만, 클릭 불가 명확 | 있음 — **전체 행 opacity** 검토 |
| P0-6 | **하단 탭** | 장바구니 수량 뱃지, 현재 탭 강조 | `BottomNav` — **대비 강화** |
| P0-7 | **장바구니** | 합계 크게, **「주방으로 주문」** 한 줄 CTA, 카운터 결제 안내 | copy·버튼 높이 48px+ |
| P0-8 | **접수 후** | 주문번호·진행(접수→조리→준비)·「이 폰에서만 확인」 | `OrderProgressSteps` + copy — **간격·색** |

### P0 — 점주 `/m/{tenant}/menus`

| # | 항목 | 한국 관습 | 목표 |
|---|------|-----------|------|
| P0-M1 | **손님과 같은 리스트** | 점주가 보는 = 손님이 보는 | 카탈로그 카드 = `MenuBoard` 토큰 공유 |
| P0-M2 | **미리보기** | KO/EN/日/中 탭으로 손님 화면 새 탭 | `MerchantConsumerPreviewLinks` — 버튼 위계 정리 |
| P0-M3 | **저장 피드백** | 저장됨/실패 토스트·배너 명확 | 기존 `?ok=saved` — 색·위치 통일 |

### P1 — 신뢰·완성도

| # | 항목 | 비고 |
|---|------|------|
| P1-1 | 메뉴 상세: 큰 사진, 옵션 라디오 터치 영역, 하단 고정 「담기」 | `menu/[itemId]` |
| P1-2 | 빈 장바구니·빈 주문 목록 **일러스트/문구** 친절하게 | |
| P1-3 | 로딩: 스켈레톤 또는 짧은 `role=status` (한국어/선택 언어) | |
| P1-4 | 목록형 `/barrier-free` 링크는 유지하되 **보조 링크** 톤 (본 메뉴가 주) | |
| P1-5 | 다크모드: **라이트 우선** 검수 후 dark 깨짐만 수정 (영업은 라이트) | |

| P2-2 | 매장 **InfoBanner**(셀프바·알레르기) | 테넌트 설정 필드 |

## 3. 화면별 “한국형” 와이어 기대치 (텍스트 스펙)

영업 때 “이 정도면 익숙하다”고 말할 수 있는 **최소 레이아웃**입니다.

```
┌─────────────────────────────┐
│ [테이블 12]  가게명    [KO EN 日] │  ← 헤더
├─────────────────────────────┤
│ ①메뉴 ②장바구니 ③주방          │  ← FLOW (선택)
│ [전체][메인][사이드]… →스크롤   │  ← 카테고리 칩
├─────────────────────────────┤
│ [img] 메뉴명          [담기] │
│       설명 한 줄              │
│       12,000원                │
│ ─────────────────────────── │
│ [img] 품절메뉴        [품절] │
├─────────────────────────────┤
│  메뉴    장바구니²   주문      │  ← 하단 탭
└─────────────────────────────┘
```

**장바구니**

```
합계 24,000원  (크게)
[ 카운터 결제 안내 박스 ]
테이블 / 요청사항
[ ███ 주방으로 주문 보내기 ███ ]
```

## 4. 디자인 토큰 (코드 기준 — 건드릴 때만)

| 토큰 | 값 | 용도 |
|------|-----|------|
| `chaya-primary` | `#a43700` | CTA·선택 칩 |
| `chaya-bg` / `chaya-surface` | `#fcf9f8` / `#fff` | 배경·카드 |
| 터치 최소 | 44px | 버튼·칩·탭 |
| 주 CTA | 48px+, `rounded-2xl`, full width | 주문·담기(상세) |
| 본문 | 14~16px, 메뉴명 semibold | 가독성 |

**하지 말 것:** primary 색 남발(전 버튼 주황), 12px 이하 본문, 담기와 링크 구분 없음.

## 5. 영업 10분 — UI만 볼 때

1. 손님 폰 라이트 모드, `?lang=en` → **메뉴가 “앱 같다”** 느낌인지  
2. 담기 2번 → 장바구니 **합계·주황 CTA**  
3. 점주 폰 메뉴 관리 → **손님과 같은 줄** + EN 미리보기  
4. 품절 토글 → 손님 화면 **품절 UI** (새로고침 후)

UI 패스 P0 통과 = 위 4장면 스크린샷을 영업 자료에 넣을 수준.
