# 점주 화면 UX 단계 정리 (1~6)

점주 `/m/*` 화면에서 개발·운영용 문구를 숨기고 점주용 카피·흐름으로 맞추는 작업.  
**6단계까지 끝난 뒤** 아래 「공사(구조 개편)」 백로그를 사용자에게 **리마인드**할 것 (2026-05-22 요청).

## 진행 표

| 단계 | 화면 | 상태 | 비고 |
|------|------|------|------|
| 1 | 대시보드 | ✅ | 지금 할 일, 내부 배너 숨김 |
| 2 | 주문 큐 | ✅ | #주문번호, 개발자 오류 숨김 |
| 3 | 테이블 QR | ✅ | 헤더·문구 정리 |
| 4 | 메뉴 관리 | ✅ | 추가 폼 상단, 고급 접기 |
| 5 | 운영 체크 (`/readiness`) | ✅ | 점주 오픈 전 체크, runbook은 내부 UI만 |
| 6 | 서브내비·공통 | ✅ | 주문·테이블·메뉴 우선, 더보기, 공통 헤더 |

내부 안내만 다시 보려면 Vercel에 `NEXT_PUBLIC_MERCHANT_INTERNAL_UI=true` (점주 프로덕션에는 비권장).

## 6단계 완료 후 리마인드 — 공사(구조 개편) 백로그

문구·순서만으로는 부족하다고 합의한 항목. **기능은 유지**, 정보 구조·화면 분리·모바일 조작성 중심으로 별도 스프린트 권장.

### 1. 메뉴 관리 — 수정 UX

- **문제:** 메뉴마다 `details` 안에 긴 수정 폼이 붙어 있어, 메뉴가 많으면 스크롤·인지 부하가 큼.
- **1차 반영 (2026-05):** 목록 = 가격 저장·품절·자세히 수정 링크 → `/m/{tenant}/menus/{menuId}` 전용 편집. 추가 폼은 목록 상단 유지.
- **2차 반영 (2026-05):** 메뉴 목록 **품절만** 필터 (`?sold_out=1`), 오늘 허브에서 품절만 보기 링크.
- **3차 반영 (2026-05):** 자세히 수정 **탭** (기본 · 사진 · 옵션·번역), 목록 **가격 변경 시에만** 저장 버튼.
- **남음(선택):** 편집 탭 URL·리다이렉트 다듬기.

### 2. 점주 내비·우선순위

- **반영 (2026-05):** 하단 탭 `오늘 · 주문 · 메뉴 · 분석` — 설정·테이블은 `/m/{tenant}/more`.
- **남음(선택):** 메뉴 권한 없을 때 손님 메뉴 탭 라벨 정리.

### 6. 운영 vs 분석 분리 (2026-05)

- **오늘** (`/dashboard`): B안 홈 — 운영·메뉴·오늘 매출 카드, 상단 ≡/🔔, 주문 CTA (`docs/MERCHANT_HOME_WIREFRAME.md`).
- **분석** (`/analytics`): 7·30일 매출 막대, 시간대·요일·인기 메뉴 차트.
- **설정** (`/more`): 테이블 QR, 운영 체크, 푸시 알림, 로그아웃.

### 3. 모바일·한 손 사용 — 주문 큐

- **1차 반영 (2026-05):** 테이블 제거 → **주문 카드** + 다음 단계 **큰 버튼** (접수·조리·서빙·완료·취소).
- **2차 반영 (2026-05):** 「전체」 보기에서 **완료·취소** 주문 `<details>` 접기.
- **3차 반영 (2026-05):** 홈과 동일 **4탭** + 티켓 카드 (`docs/MERCHANT_ORDERS_WIREFRAME.md`).
- **남음:** 구역 칸반, 일괄 접수, 스와이프.

### 4. 운영 체크 vs 점주 일상

- **문제:** `/readiness` 등에 runbook·파일명·스크립트 이름이 섞이면 점주에게는 노이즈.
- **방향:** 점주용 체크리스트(할 일 문장) vs 운영자 runbook 문서 분리; 5단계에서 문구 정리 후에도 구조 분리 검토.

### 5. 통합 「오늘」 허브

- **1차 반영 (2026-05):** `/m/{tenant}/dashboard` — `MerchantTodayHub` (대기·품절·테이블 미설정 알림). 하단 탭 라벨 「오늘」.
- **2차 반영 (2026-05):** 품절만 필터 링크, 테이블 DB 로드 실패 시 오늘 허브 안내 카드.
