# 점주 홈(오늘) 화면 — 텍스트 와이어 (B안)

**경로:** `/m/{tenant}/dashboard` · 하단 탭 **「오늘」**  
**목적:** 영업 중 “지금 손대야 할 일”을 한 화면에서 보고, 숫자 탭으로 주문·메뉴·분석으로 바로 이동.  
**벤치마크 참고:** 네이버 스마트스토어 홈의 **카드·접기·갱신·숫자=링크** (마케팅·정산·리뷰 항목은 이식하지 않음).

**상태 묶음 규칙 (5단계 — 확정, 2026-05)**

| 홈 라벨 | DB `orders.status` | 집계 범위 |
|---------|----------------------|-----------|
| **새 주문** | `pending` | 미처리 큐 (전체) |
| **조리중** | `accepted`, `preparing` | 주방 큐 (전체) |
| **서빙완료** | `ready` | 나가기 전 (전체) |
| **결제완료** | `completed` | **당일 KST** |
| **취소** | `cancelled` | **당일 KST** |

**레이아웃:** 1행 2칸(새 주문·조리중) + 2행 3칸(서빙·결제·취소, compact).

**취소 사유 (백로그):** 손님 변심·재료 품절·중복 주문·직원 실수·테이블 이동·앱 오작동 등 — DB·UI는 별도 스프린트.

**「완료」단어 미사용:** 점주 혼동(조리 완료 vs 결제 완료) 방지 → **서빙완료** / **결제완료**.

주문 **상세 단계·큰 버튼**은 **주문 탭**(`/orders`)에서 유지. 홈은 요약만.

**「오늘」매출·주문·완료·취소 숫자 = 영업일(날짜) 기준**

- **한국 시간(KST) 당일 00:00 ~ 익일 00:00** — 롤링 “최근 24시간” **사용하지 않음**.
- 분석 탭 일별 차트의 “오늘 막대”와 **같은 날짜 경계**로 맞춤.
- 구현: `getMerchantTodayKstMetrics`, `getKstCalendarDayBounds` (`lib/orders/merchant-analytics.ts`).

## 화면 전체 구조 (위 → 아래)

```
┌─────────────────────────────────────────────┐
│ [≡]        {매장 표시명}              [🔔 N] │  ← ① 상단 바
├─────────────────────────────────────────────┤
│ (시스템 알림 1줄 — 권한·오류 등, 있을 때만)   │
│ (내부/프리뷰 배너 — 있을 때만)                │
├─────────────────────────────────────────────┤
│ ② 운영 관리 카드 (접기/펼치기, 기본 펼침)      │
├─────────────────────────────────────────────┤
│ ③ 메뉴·매장 카드 (접기, 기본 펼침)            │
├─────────────────────────────────────────────┤
│ ④ 오늘 매출 요약 카드 (접기, 기본 펼침)       │
├─────────────────────────────────────────────┤
│ ⑤ 주문 큐 바로가기 (고정 CTA)                 │
├─────────────────────────────────────────────┤
│ ⑥ 법적·고객센터 (접기, 기본 접힘)             │
└─────────────────────────────────────────────┘
│  하단 탭: 오늘 | 주문 | 메뉴 | 분석            │  ← 변경 없음
└─────────────────────────────────────────────┘
```

**모바일:** 데스크톱용 `MerchantSubnav`(상단 칩)는 **숨김** 유지.  
**데스크톱(md+):** 기존 서브내비 유지 가능.

## ① 상단 바

| 요소 | 동작 | 문구/규칙 |
|------|------|-----------|
| **≡** (햄버거) | 탭 | `/m/{tenant}/more` (설정·테이블 QR·운영 체크·로그아웃 등) |
| **매장명** | — | `branding.displayName` (짧게, 한 줄 말줄임) |
| **🔔** | 탭 | `/m/{tenant}/notifications` *(신규 라우트 또는 more 내 앵커 — 구현 시 선택)* |
| **N 뱃지** | — | `pending` 건수 > 0 이면 표시 (또는 미확인 알림 수) |

**접근성:** 알림 버튼 `aria-label="새 주문 알림, N건"`.

## ② 운영 관리 카드 (핵심)

**카드 제목:** `운영`  
**우측 헤더:** `오후 3:42 갱신` · `[⟳ 갱신]` · `[∧ 접기]`  
**갱신:** `OrderStatusRefresh` 또는 전체 `router.refresh()` — 기존 주문 폴링 재사용.

### 펼침 상태 — 본문

```
┌─ 운영 ─────────────── 오후 3:42 갱신  ⟳  ∧ ─┐
│                                              │
│  새 주문          진행 중                     │
│  ┌──────────┐    ┌──────────┐                │
│  │    3건    │    │    5건    │  ← 전체 너비 탭 영역 │
│  └──────────┘    └──────────┘                │
│  손님이 넣은 주문   접수·조리·나가기 전        │  ← 보조 한 줄 (12px)
│                                              │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│                                              │
│  오늘 완료              취소                  │  ← 2열, 시각적 비중 낮음
│     12건                   1건               │
│                                              │
└──────────────────────────────────────────────┘
```

| 칸 | 링크 (탭 시) | 강조 |
|----|--------------|------|
| **새 주문** `N건` | `/m/{tenant}/orders?status=pending` | `N>0` → 테두리/배경 강조 (빨강 계열) |
| **진행 중** `N건` | `/m/{tenant}/orders` + 클라이언트 필터 또는 `?bucket=in_progress` *(구현 시 쿼리 설계)* | 보통 강조 |
| **오늘 완료** | `/m/{tenant}/orders?status=completed` *(당일만 필터는 구현 시)* | 기본 텍스트 |
| **취소** | `/m/{tenant}/orders?status=cancelled` | 회색·작은 글씨 |

**보조 문구 (카드 하단, 선택):**  
`주문 단계 변경은 「주문」 탭에서 합니다.`

### 접힘 상태

- 제목 줄만: `운영 · 새 주문 3 · 진행 중 5` (숫자만 요약)  
- 탭하면 펼침.

## ③ 메뉴·매장 카드

**카드 제목:** `메뉴·매장`  
**우측:** 갱신 시각 · ⟳ · 접기 (운영 카드와 동일 패턴, 갱신은 공유 가능)

| 행 라벨 | 숫자 | 링크 | 표시 조건 |
|---------|------|------|-----------|
| 품절 메뉴 | `N개` | `/m/{tenant}/menus?sold_out=1` | `N>0` 시 행 강조 |
| 테이블 QR | `미설정` 또는 `N개` | `/m/{tenant}/tables` | 테이블 권한 있을 때 |
| 등록 메뉴 | `N개` | `/m/{tenant}/menus` | 메뉴 권한 있을 때 |

**테이블 DB 오류:** 숫자 대신 `불러오기 실패` + `설정에서 다시 시도` → `/more`.

**메뉴 권한 없음:** 품절·등록 메뉴 행 숨김.

## ④ 오늘 매출 요약 카드

**카드 제목:** `오늘 매출`  
**우측:** `5월 26일(수) · 오늘` · ⟳ · 접기

| 요소 | 내용 |
|------|------|
| 부제 | `{dateLabel} · 자정~자정(한국 시간)` |
| 큰 숫자 | `1,234,000원` (**당일** 접수 주문 매출 합, 취소 제외) |
| 보조 | `주문 18건` (**당일** 접수 건수, 취소 포함 여부는 와이어와 동일: 건수는 전체, 매출만 취소 제외) |
| 링크 버튼 | `분석에서 자세히 보기 →` → `/m/{tenant}/analytics` |
| 미리보기 (2차) | 7일 미니 차트 — **일별 막대**, 오늘 칸과 숫자 일치 |

**롤링 24시간 금지:** 어제 밤 주문이 “오늘 매출”에 섞이지 않도록 DB는 `created_at >= 오늘 00:00 KST` AND `< 내일 00:00 KST`.

**정산·예정 입금 등:** 넣지 않음 (PG·오프라인 정책과 맞춤).

운영 카드 아래, 스크롤 끝에 가기 전에 항상 보이게.

## 주문 탭과의 관계 (홈 건드리지 않음)

| 홈 | 주문 탭 |
|----|---------|
| 4칸 숫자 | 카드 + `접수` → `조리 시작` → `서빙 준비` → `완료` |
| `진행 중` = 3 status 합 | 목록에서 status 뱃지 그대로 표시 |
| 숫자 탭 → 필터 진입 | 필터 칩 + 완료·취소 접기 유지 |

**2차 (백로그):** 점주 설정 `간편 모드` — 버튼 2~3개로 줄이기, DB status는 유지.
