# 점주 주문 탭 — 텍스트 와이어 (1·2차)

**경로:** `/m/{tenant}/orders?tab=…` · 하단 탭 **「주문」**  
**원칙:** 홈 운영 카드와 **같은 4칸·같은 필터**. 처리 큐 = 티켓 + 다음 단계 버튼 1개.

---

## 1차 — 구조 (스프린트 1)

### 제거
- 상단 3칸 요약 (신규·조리 중·진행 매출) — **홈과 중복**
- 8개 상태 칩 (`전체` + `진행 중` + 6 status)
- 큰 헤더 + 「← 홈」 (하단 탭 있음)
- 안내 문구 「카드의 큰 버튼으로…」

### 상단 (sticky)
```
[로고][매장명]                    [⟳][🔔][≡]   ← 홈과 동일 높이
────────────────────────────────────────────
[ 새 주문 3 ] [ 진행 중 5 ] [ 완료 12 ] [ 취소 1 ]   ← sticky, 홈과 동일 라벨
```

| tab | 쿼리 |
|-----|------|
| `pending` | `status=pending` |
| `cooking` | `bucket=cooking` (`accepted`+`preparing`) |
| `ready` | `status=ready` |
| `paid` | `status=completed&today=1` (KST) |
| `cancelled` | `status=cancelled&today=1` (KST) |

탭 UI도 홈과 동일 **2+3** 그리드.

- `tab` 없이 진입: **pending > 0 → `pending`**, 아니면 **`in_progress`**
- 구 URL `?status=` / `?bucket=` → 동일 tab으로 해석 (호환)

### 본문
- 선택 탭 주문만 리스트 (완료·취소 접기 **없음** — 탭이 분리)

---

## 2차 — 티켓 카드 (스프린트 2)

```
┃ 12번 테이블          24,000원
┃ #1042 · 5분 전
┃ 불고기덮밥 ×2, 콜라 ×1
┃ 요청: 덜 맵게
┃ ┌─────────────────────┐
┃ │      접수하기        │  ← primary 전폭
┃ └─────────────────────┘
┃            주문 취소     ← danger 텍스트/작은 버튼
```

- **왼쪽 색 띠** = status (대기 빨강, 조리 주황, 준비 초록…)
- **5분 전** — 15분↑ 주황, 25분↑ 빨강 (진행 중만)
- 메뉴 2줄 + `외 N개`
- 버튼 1개 primary; 취소만 보조

## 구현 체크리스트

- [x] 1차 — 5탭 sticky(2+3), 요약·8칩 제거, 홈 헤더, `?tab=`
- [x] 2차 — 티켓 카드, items, 색 띠, 대기 경고, 전폭 primary
- [x] 2b — 5단계 버튼·뱃지, 조리 탭 구역, 상태 후 탭 이동, 오래된 순 정렬
- [x] 3차 — 일괄 접수, 취소 사유(`orders.cancel_reason`), 간편 모드(기기 저장)
