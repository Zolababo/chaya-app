# MVP 영업 준비 기준 (2026년 6월)

점주 영업·실매장 시연 전 **「이 정도면 데모·파일럿 가능」** 판단용 체크리스트입니다.  
앱 경계: **손님 `/t` · 점주 `/m` · 운영 `/ops`** 는 `apps/consumer-menu` 단일 배포. **SNS 앱**은 별도 저장소/일정이면 본 문서 §6 참고.

---

## 1. 손님 — 주문 MVP (필수)

- [ ] QR `/t/{tenant}` 진입 후 **언어 선택** 가능 (ko + en, ja, zh-Hans, zh-Hant, vi, th, ru)
- [ ] 메뉴·상세·장바구니·주문 제출·주문 현황 **끝까지** (카운터 결제 안내)
- [ ] 번역 없는 메뉴는 **한국어 이름으로 폴백**
- [ ] `pnpm smoke:consumer` 프로덕션 PASS

## 2. 점주 — 운영 MVP (필수)

- [ ] 메뉴 목록이 **손님 화면과 같은 형태**로 미리보기
- [ ] 사진·가격·품절·**다국어 이름·설명** 입력
- [ ] 「손님 메뉴판 열기」·언어별 미리보기 (`?lang=`)
- [ ] 신규 주문이 `/m/{tenant}/orders` 에 표시

## 3. 접근성 — 당장 쓸 수 있음 (필수)

- [ ] TalkBack/VoiceOver로 **메뉴 → 장바구니 → 주문** 1회 (선택 언어)
- [ ] `docs/CONSUMER_C2_MANUAL_RUNBOOK.md` 3·4단계 통과
- [ ] `npm run a11y:baseline -- --tenant demo` PASS

## 4. 운영 `/ops` (필수)

- [ ] 매장 초대·멤버·`GET /health` guestOrderRpcsProbe 정상

## 5. 영업 데모 시나리오 (10분)

1. 손님 폰: 영어 → 메뉴 2개 담기 → 주문  
2. 점주 폰: 주문 접수 → 품절 토글 → 손님 화면 반영 확인  
3. 일본어/중국어 간체로 메뉴명 1개만 바꿔 재확인  

## 6. SNS 앱 (동일 기간 목표 — 별도 트랙)

소비자·점주와 **같은 Supabase·tenant_slug** 를 쓰는 경우:

- [ ] 매장 프로필·메뉴 링크가 `/t/{tenant}` 로 연결
- [ ] 영업 자료에 SNS는 「홍보 채널」、주문은 CHAYA 메뉴판으로 통일
