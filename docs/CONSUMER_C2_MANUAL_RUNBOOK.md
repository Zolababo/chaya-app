# C2 수동 동선·접근성 런북

자동 1~2단계는 `docs/CONSUMER_C2_VERIFICATION.md` · `pnpm smoke:consumer` 참고.  
아래는 **실기기**에서 한 번씩 체크할 때 쓰는 순서입니다 (`tenant` 예: `demo`).

**프로덕션:** https://chaya-app.vercel.app

---

## 3단계 — 주문 동선 (~10분)

| # | 할 일 | URL·기준 |
|---|--------|----------|
| 1 | 메뉴 **담기** (목록 또는 상세) | `/t/demo` → 행 **담기** 또는 상세 **장바구니에 담기** |
| 2 | 장바구니·카운터 안내 | `/t/demo/cart` — 「결제는 매장 카운터」, 「주방으로 주문 보내기」 |
| 3 | 접수·진행 단계 | 제출 후 `/t/demo/orders/{id}` — 진행 단계·상태 |
| 4 | 주문 탭·폴링 | 하단 **주문** → 목록 → 상세, 「최신 상태로」 |
| 5 | 주소 복사 | 접수 화면 **이 주문 주소 복사** (선택) |
| 6 | 세션 제한 | 시크릿 창에서 같은 주문 URL → 「주문을 찾을 수 없습니다」 |

점주 확인: `/m/demo/orders` 에 주문 표시.

---

## 4단계 — 접근성 (~15분, 기기 1종)

| # | 할 일 | 통과 기준 |
|---|--------|-----------|
| 1 | VoiceOver / TalkBack | 하단 탭·메뉴·장바구니·주문이 **한국어**로 읽힘 |
| 2 | 키보드만 (PC) | **본문으로** → 카테고리 칩 → 담기 포커스 |
| 3 | 동작 줄이기 | iOS/Android **애니메이션 줄이기** 켠 뒤 주문 상세 — 자동 폴링 안내·수동 버튼 |

이슈는 짧게 메모만 남기고 이후 티켓으로.

---

## 완료 표기

- `docs/MERCHANT_OPERATIONS_ROADMAP.md` 소비자 C2 비고에 「수동 완료 일자」 추가
